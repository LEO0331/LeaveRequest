# Leave Management System

Interactive Leave Management System
An interactive web application built with modern technologies to manage, filter, and load-test employee leave requests.

🚀 Technologies Used

* React (v18.2.0) with Vite: Leveraged for a dynamic, high-performance UI and efficient development workflow.
* TypeScript: Ensures type safety and improves maintainability across the codebase.
* MUI (Material UI): Provides a consistent design language and a robust library of pre-built components for rapid styling.
* Faker: Used to generate 10,000+ records of random data for realistic load testing and performance benchmarking.
* Local Storage: Acts as a mock backend/browser-based database for data persistence.

📋 Task Details & FunctionalityHome Screen
The primary dashboard includes:

* Leave Request List: A comprehensive view of all current requests.
* Search & Filtering: A global search bar for keywords and specific column filters for Start/End dates and Users.
* Sorting: Ability to sort columns for better data organization.
* Details View: Click any request to view full details or enter Edit Mode.

Leave Request Form
Accessible via the "Add New Leave Request" button or via the edit action. The form includes:

* Date Pickers: For selecting Start and End dates.
* Leave Type Dropdown: Options include Personal, Sick, Vacation, and Bereavement.
* Dynamic Day Calculation: Automatically calculates the duration, rounded to two decimal places using the floor function (e.g., 4.376 → 4.37).
* Reason Text Area: A field for justification, limited to 50 characters.
* User Selection: A dropdown containing a dummy list of ~10 users.

Validation Logic
The application performs real-time validation upon clicking Save. Indicators are displayed if:

* The request overlaps with an existing leave period.
* The Start Date is in the past.
* The End Date is before the Start Date.
* The Reason field is empty.
* The calculated duration is zero.
* No Leave Type or User is selected.

✨ Main Features

* Mock Backend: Uses Browser Local Storage to simulate database interactions.
* Grouping & Filtering: Grouping by client and advanced filtering capabilities.
* Load Testing: Smoothly handles 10,000 fake entries generated by Faker.
* State Management: Includes loading states during submission and success notifications upon completion.
